## WS 持续发送与重连噪音修复
- 2026-03-25：定位到详情页 `useInspectorLogStream` 依赖包含 `onRecord/onStatusChange`，在 React 渲染中回调引用变化会触发 effect 重跑，导致 `client.stop()/start()`、重复 `hello` 握手与“看起来一直在发送”。
- 决策：WS 生命周期仅由 `baseUrl + identity` 驱动；回调引用通过 `useRef` 热更新，避免重连同时保持回调最新。
- 验证：新增 `tests/useInspectorLogStream.test.tsx`（2 条）覆盖“仅回调变化不重连”和“不重连且命中最新回调”；并回归 `tests/ws.test.ts`，均通过。

## WS 通知 + HTTP 增量拉取
- 2026-03-25：详情页日志同步策略从“定时轮询 + WS”改为“WS 仅通知，HTTP 负责增量拉取”。
- 行为：初次加载仍走 `GET /v2/logs?waitMs=0&limit=200`；收到 `event.log_record` 后触发 `GET /v2/logs?afterId=<lastRecordId>&waitMs=0&limit=200`。
- 同步控制：对 WS 高频通知做串行合并（in-flight + queued），避免并发重复拉取。
- 验证：新增 `tests/ui/ClientDetailPage.test.tsx`，覆盖“无通知不拉取、通知后按 afterId 增量拉取”；测试通过。
