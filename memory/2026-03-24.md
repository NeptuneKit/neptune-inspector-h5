# 2026-03-24

- `neptune-inspector-h5` 新增 `sources` 和 `metrics` 面板，分别对接 `GET /v2/sources`、`GET /v2/metrics`。
- `logs` 面板保留 `afterId + waitMs` 长轮询增量拉取。
- 新增 Inspector WS 双向链路：启动后自动连接 `/v2/ws`，发送 `hello(role=inspector)`、15s heartbeat、指数退避重连，并在前端提供 `command.send(ping)` 下发与 ACK 观察区。
- `command.send(ping)` 在前端先校验 target 至少包含 `platforms | appIds | sessionIds | deviceIds` 中任一非空项。
- 增补了 `logs / sources / metrics / ws` 的 helper 测试，并通过 `npm test` 与 `npm run build`。
- 经过网络与协议审计后，保留浏览器原生 `fetch` / `URL` 作为 HTTP 基础设施，并在响应边界引入 `zod` 做运行时 JSON 校验，避免自研解析层。
- 为 `neptune-inspector-h5` 补齐 GitHub Actions CI：`push/main` 和 `pull_request` 统一执行 `npm ci`、`npm test`、`npm run build`，README 增补了 CI 说明。
- 为 macOS app 资源打包补齐统一入口 `scripts/build-desktop-assets.sh`，CI 通过该脚本产出标准 `dist/`，测试通过后再上传 artifact，供 `neptune-desktop-macos` 和后续发布流水线复用。
